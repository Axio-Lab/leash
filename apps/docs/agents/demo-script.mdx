This is the script the `pnpm --filter @leash/mcp test:yc-demo-devnet`
integration test exercises. It runs against live Solana devnet
infrastructure — no mocks, no faucets to hit manually.

## Prerequisites

- A reachable Leash API. The hosted one at
  `https://api.leash.market` works out of the box; if you're
  self-hosting, set `LEASH_API_URL` on the MCP entry.
- A Solana devnet RPC endpoint with reasonable headroom. The default
  public cluster works for the demo; for recordings, point to a
  Helius / Triton endpoint via `LEASH_RPC_URL`.
- A side terminal with the `solana` CLI for the devnet airdrop step
  (or a faucet bookmark — faucet.solana.com / quicknode.com/faucet/sol
  also work). The Leash protocol no longer ships a server-side faucet;
  every executive keypair is funded by the operator.

## Setup (one-time)

Wire the standalone MCP into your AI host (Cursor shown):

```jsonc
// ~/Library/Application Support/Cursor/User/settings.json (or equivalent)
{
  "mcpServers": {
    "leash": {
      "command": "npx",
      "args": ["-y", "@leash/mcp"],
    },
  },
}
```

Restart the host. `tools/list` should now expose all 11 `leash_*`
tools. If it doesn't, run `leash-mcp doctor` in a side terminal and
confirm `api_check: ok`.

## The demo (90-120 seconds on camera)

Each beat below is one prompt the operator types into the AI host;
the next line is the tool the agent ends up calling.

### 0. (Pre-roll: empty config)

> _"What's my Leash balance?"_

The agent calls `leash_check_treasury_balance`, gets back
`status: no_agent` with a clean recovery message. **Don't skip this
beat** — it's what makes the next step land.

### 1. Onboard, step 1 — generate the executive

> _"Register me a Leash agent please."_

The agent calls `leash_register_agent` with no arguments. The MCP:

- Generates a fresh ed25519 keypair locally.
- Persists it to `~/.config/leash/agent.json` under
  `pending_register` (chmod 600).
- Reads the executive's SOL balance via RPC (zero — fresh keypair).
- Returns `status: "funding_required"` with the pubkey and a
  numbered list of funding instructions.

The agent surfaces the executive pubkey + the SOL amount needed
(0.01 by default). On screen, switch to the side terminal:

```bash
solana airdrop 1 <executive_pubkey> --url https://api.devnet.solana.com
```

### 2. Onboard, step 2 — finish the mint

> _"OK, the executive is funded. Continue."_

The agent calls `leash_register_agent` again with no arguments. The
MCP:

- Re-checks the balance (now ~1 SOL).
- Mints the MPL Core agent asset (`createAgent`).
- Sets unlimited USDC delegation from the treasury PDA to the
  executive (`setSpendDelegation`, capped at `u64::MAX`).
- POSTs `/v1/agents/record` so the API has a row for receipts +
  discovery + reputation.
- Promotes `pending_register` to a full `agent.json` and hot-swaps
  the in-memory host — no MCP restart.

The agent surfaces the `agent_mint`, treasury, both tx signatures,
and a Solscan link.

### 3. Identity

> _"Who am I now?"_

The agent calls `leash_get_identity`. Returns mint, treasury,
executive pubkey, network. Show the Solscan link in passing.

### 4. Fund the treasury (optional, devnet)

> _"Send $1 USDC to my treasury so I can pay for things."_

The treasury starts empty — the protocol no longer auto-funds it.
On devnet, send a few USDC test tokens from any source to the
agent's treasury PDA. On mainnet, do the same with real USDC. The
treasury address is the `treasury` field from step 3 (or
`leash_get_identity`).

### 5. Balance

> _"What's my balance?"_

`leash_check_treasury_balance`. Shows the USDC just funded and the
SOL the executive holds. **This is the moment that lands** — the
agent went from "I have nothing" to "I have a fully-controlled
on-chain wallet that I can spend from" without the operator
touching a wallet UI mid-demo.

### 6. Discover

> _"Find me a paid OCR service under 10 cents."_

`leash_discover` with `{ capability: 'ocr', max_price_usdc: 0.10 }`.
Returns approved listings.

### 7. Vet

> _"What's the seller's reputation?"_

`leash_reputation` against the seller's agent mint. Returns
`settled_calls`, `dispute_rate`, normalised rating in `[0, 1]`.

### 8. Pay

> _"Pay this link: https://leash.market/x/abc123"_

`leash_pay_payment_link`. Probes the x402 paywall, signs the SPL
transfer locally, settles. Returns a real `tx_signature` and
`receipt_hash`.

### 9. Withdraw

> _"Withdraw 50 cents USDC to YOUR_WALLET_ADDRESS."_

`leash_withdraw_treasury`. Real on-chain transfer via
`mpl-core::Execute`. Returns the `tx_signature` and a Solscan link.

## What to highlight on camera

1. **No browser is open** during the demo — everything runs in the
   AI host's chat and the terminal.
2. **The Solscan link works on every step** — every state change is
   on-chain and verifiable.
3. **The same agent works in the chat product** — open
   [`agent.leash.market`](https://agent.leash.market) and the
   freshly-minted agent shows up.
4. **The CLI works against the same identity** — `leash agent show`,
   `leash treasury balance` from the terminal.

## Recording tips

- Use a 1080p screen recording at 30 fps. The MCP outputs are
  tight — readability matters more than frame rate.
- Increase the AI host's font size before recording so receipts and
  signatures are legible.
- Pre-load Solscan in a side tab so you can pivot to it after step 6
  without dead air.
