---
title: 'Run it locally'
description: Boot the API, indexer, webhook worker, and explorer on your own machine.
---

The Leash backend is four pieces. They all live in the same monorepo
and talk to each other via Turso/SQLite + Redis (with sane in-memory
fallbacks for dev).

| Piece               | Workspace                    | Default port | Required for                                         |
| ------------------- | ---------------------------- | ------------ | ---------------------------------------------------- |
| **API server**      | `apps/api` → `pnpm dev`      | `8801`       | Everything below — `prepare`/`submit`, receipts, OpenAPI. |
| **Indexer worker**  | `apps/api` → `pnpm indexer:dev` | n/a       | Chain → events ingestion + receipt-pull worker.       |
| **Webhook worker**  | (in-process with `apps/api`) | n/a          | Auto-runs in `pnpm dev`; standalone in prod.          |
| **Explorer UI**     | `apps/explorer` → `pnpm dev` | `3100`       | Browser-facing protocol explorer.                     |

The runner (`apps/web` playground sidecar) is separate and documented
in [Quickstart](/quickstart) — these instructions cover the new API
+ explorer + indexer surfaces shipped in v0.2.

## 1. Install + build once

```bash
pnpm install
pnpm turbo run build typecheck lint test
```

## 2. Bring up the API server

```bash
cd apps/api
pnpm dev
```

```text
[leash-api] dev server on http://0.0.0.0:8801
[leash-api] OpenAPI:   http://0.0.0.0:8801/openapi.json
[leash-api] webhook delivery worker started (every 2s)
```

`pnpm dev` runs `tsx ./src/dev.ts` — that includes the in-process
webhook worker, so you don't need a separate process for that surface
in dev. SQLite lives at `./.leash-api.db` by default.

### Required env (with sane defaults)

```bash
export LEASH_API_HOST=0.0.0.0           # default 0.0.0.0
export LEASH_API_PORT=8801              # default 8801
export LEASH_API_DB_URL=file:./.leash-api.db   # or libsql://… for Turso
export LEASH_API_RPC_DEVNET=https://api.devnet.solana.com
export LEASH_API_RPC_MAINNET=https://api.mainnet-beta.solana.com
export LEASH_API_REDIS_URL=redis://localhost:6379   # optional; in-memory fallback if unset
export LEASH_API_RATELIMIT_RPM=120
```

### Mint a local API key

There is no self-service issuance yet (see [Roadmap](/api/roadmap)).
Two options for local dev:

- Set `LEASH_API_BOOTSTRAP_KEY=lsh_test_dev_$(uuidgen)` (and
  optionally `LEASH_API_BOOTSTRAP_KEY_LABEL=local`) before
  `pnpm dev`. The key is auto-registered on first boot for whichever
  network the prefix encodes.
- Or `node -e "import('@leash/api/src/storage/api-keys.js').then(({ createApiKey }) => createApiKey(...).then(console.log))"`
  inside a one-off script. The plaintext returned is the only time
  it's exposed.

### Verify

```bash
curl http://localhost:8801/v1/health        # {"ok":true,"version":"0.1.0"}
curl http://localhost:8801/openapi.json | head -20

# Authenticated
curl -H "Authorization: Bearer $LEASH_API_KEY" http://localhost:8801/v1/version
```

## 3. Bring up the indexer worker

The indexer is a separate long-running process that turns on-chain
agent activity into rows in the same `events` table the API writes.

```bash
cd apps/api
pnpm indexer:dev          # tsx ./src/indexer/cli.ts
# or in prod:
pnpm build && pnpm indexer
```

```text
[leash-indexer] tick devnet  watchlist=0 cursors=0 events_written=0
[leash-indexer] tick mainnet watchlist=0 cursors=0 events_written=0
```

The watchlist is empty until the API sees its first agent — every
`prepare*` call and every receipt ingest auto-adds the agent +
treasury PDA to the watchlist. Then the next indexer tick starts
backfilling.

### Required env (in addition to the API server's)

```bash
export LEASH_API_INDEXER_INTERVAL_MS=5000     # default 5s
export LEASH_API_INDEXER_PAGE_SIZE=100        # max 1000
export LEASH_API_NETWORKS=devnet,mainnet      # default both
```

Both indexer + API talk to the same DB and the same RPC env vars,
so nothing extra is needed — the indexer just needs to be able to
read+write the same `LEASH_API_DB_URL`.

### Verify

```bash
curl -H "Authorization: Bearer $LEASH_API_KEY" \
  http://localhost:8801/v1/indexer/status
```

```json
{
  "network": "solana-devnet",
  "watchlist_size": 0,
  "cursors": { "total": 0, "last_run_at": null },
  "events_last_hour": {}
}
```

`last_run_at` will populate after the first tick. See
[Indexer](/api/indexer) for the full state model.

## 4. Bring up the explorer UI

```bash
cd apps/explorer
pnpm dev          # http://localhost:3100
```

The explorer is a thin Next.js app — every fetch is a server action
that calls the API with read-only API keys. Browser never holds the
keys.

### Required env

```bash
export LEASH_API_URL=http://localhost:8801                   # default https://api.leash.market
export LEASH_EXPLORER_API_KEY_DEVNET=lsh_test_explorer_...   # for Devnet toggle
export LEASH_EXPLORER_API_KEY_MAINNET=lsh_live_explorer_...  # for Mainnet toggle
```

You can issue any read-only `lsh_test_*` key for the devnet slot and
any `lsh_live_*` key for the mainnet slot — there's no separate
"explorer key" type; the explorer just keeps two slots.

### Verify

Open [http://localhost:3100](http://localhost:3100). The home page
should render with the network toggle in the top right and an empty
state until traffic flows through your API. Search a pubkey or a
signature you know exists.

## 5. End-to-end smoke flow

With API + indexer + explorer all running, drive a real prepare →
submit → confirm cycle:

```bash
# 1. Prepare a delegation
EVENT=$(curl -s -X POST http://localhost:8801/v1/agents/$MINT/delegation/prepare \
  -H "Authorization: Bearer $LSH_TEST_KEY" \
  -d '{ "executive": "'$EXEC'", "stable_symbol": "USDC", "allowance": "100" }')

EVENT_ID=$(echo "$EVENT" | jq -r .event_id)
SIGNED=$(node ./sign.mjs "$(echo $EVENT | jq -r .transaction.full_base64)")

# 2. Submit
curl -X POST http://localhost:8801/v1/submit \
  -H "Authorization: Bearer $LSH_TEST_KEY" \
  -d "{ \"event_id\": \"$EVENT_ID\", \"transaction\": \"$SIGNED\" }"

# 3. Watch it land
curl http://localhost:8801/v1/events/$EVENT_ID -H "Authorization: Bearer $LSH_TEST_KEY"

# 4. Open the explorer
open "http://localhost:3100/event/$EVENT_ID"
```

Within ~10 seconds the indexer tick should pick up the matching
on-chain transaction (because the API auto-watchlisted the agent on
the prepare call), write a `confirmed` event, and the explorer view
will show the timeline.

## 6. Hook up webhooks (optional)

If you want to test the webhook surface end-to-end:

```bash
# Receiver — anything that responds 200 works. Webhook.site / ngrok / a Hono route.
ngrok http 4000

# Subscribe
curl -X POST http://localhost:8801/v1/webhooks \
  -H "Authorization: Bearer $LSH_TEST_KEY" \
  -d '{ "url": "https://abc123.ngrok.app/leash-hook", "events": [] }'

# Watch deliveries
curl http://localhost:8801/v1/webhooks/$WEBHOOK_ID/deliveries \
  -H "Authorization: Bearer $LSH_TEST_KEY"
```

Run another `prepare` + `submit` and you'll see deliveries land in
the deliveries list with `attempts: 1, delivered: true, last_status: 200`.
See [Webhooks](/api/webhooks) for the full signature scheme.

## Production layout

For staging / prod, run the four pieces as separate processes:

| Process            | Container / Service                                              |
| ------------------ | ---------------------------------------------------------------- |
| API server         | `node dist/cli.js` (or the published `leash-api` binary).        |
| Indexer worker    | `node dist/indexer/cli.js` (or the published `leash-indexer`).   |
| Webhook worker    | A second `node dist/cli.js` process with a flag to disable serving and only run the worker (or keep it in-process for low-traffic deployments — `startWebhookWorker` is exported from `@leash/api`). |
| Explorer          | `next start -p 3100` behind your CDN of choice.                  |

Point them all at the same Turso DB + the same Redis, give the API
node a Helius key (or any commercial RPC) for both networks via
`LEASH_API_RPC_DEVNET` / `LEASH_API_RPC_MAINNET`, and you're done.

For a one-shot dockerised local run that brings up runner + seller
demo + playground (the original v0.1 stack), see [Quickstart](/quickstart).
