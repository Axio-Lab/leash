`@leashmarket/sdk` is the thin TypeScript HTTP client for the public Leash
API. It is the **remote control** for a deployed Leash server — not a
buyer or seller in itself.

```text
┌────────────────────────┐  HTTP (X-Leash-Sig or API key)
│ your app, browser,     │ ─────────────────────────────► api.leash.market
│ edge fn, Bun script    │                                 │
└────────────────────────┘                                 ▼
                                                     Solana RPC
```

If you need to **sign Solana transactions locally** (real x402 paying,
seller middleware, on-chain mint/withdraw flows), use the engine-room
kits instead:

| Need                                        | Reach for                                                                                 |
| ------------------------------------------- | ----------------------------------------------------------------------------------------- |
| Pay an x402 link                            | [`@leashmarket/buyer-kit`](/sdk/buyer-kit) or [`@leashmarket/mcp`](/agents/mcp)'s `pay()` |
| Monetise an HTTP endpoint                   | [`@leashmarket/seller-kit`](/sdk/seller-kit)                                              |
| Mint / register / withdraw from agents      | [`@leashmarket/registry-utils`](/sdk/registry-utils)                                      |
| Discover, vet, manage receipts and webhooks | **`@leashmarket/sdk` (this page)**                                                        |

## Install

```bash
pnpm add @leashmarket/sdk
```

The package is **ESM-only**. Zero runtime deps beyond Umi (only used
to sign the `X-Leash-Sig` header).

## Three auth modes

```ts
import { LeashClient } from '@leashmarket/sdk';

// 1. Anonymous — public reads only.
const pub = new LeashClient({ baseUrl: 'https://api.leash.market' });

// 2. Agent-signed — webhooks and any future X-Leash-Sig endpoints.
const signed = new LeashClient({
  agentMint: 'AjfeyP...',
  executiveSecretBase58: process.env.LEASH_EXECUTIVE_KEY!,
});

// 3. Legacy API key — payment-link CRUD, receipts.
const keyed = new LeashClient({ apiKey: process.env.LEASH_API_KEY! });
```

You can pass more than one — the client picks the right mode per call.

## Methods

### Public reads (no auth)

| Method                                                                       | Endpoint                                                                               |
| ---------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| `discover({ capability?, max_price_usdc?, pricing_type?, source?, limit? })` | `GET /v1/discover`                                                                     |
| `reputation({ agentMint, network? })`                                        | `GET /v1/agents/{mint}/reputation`                                                     |
| `recordAgent(input)`                                                         | `POST /v1/agents/record` (idempotent record after client-side mint)                    |
| `paySkillsProvider(fqn)`                                                     | `GET /v1/discover/pay-skills/{fqn}` — expand a pay-skills item into its paid endpoints |

`discover()` returns a merged feed of two catalogues, distinguished by a per-item
`source: 'leash' | 'pay-skills'` field:

- `'leash'` — agents listed on the Leash marketplace (carry `seller_wallet`,
  `tools[]`, on-chain identity).
- `'pay-skills'` — providers in the Solana Foundation
  [`pay-skills`](https://github.com/solana-foundation/pay-skills) registry — the
  same catalogue the pay.sh CLI reads. These have `seller_wallet: null`,
  `tools: []`, and `rating: null`, but every entry is x402-payable today via
  `@leashmarket/buyer-kit` (and `@leashmarket/mcp`'s `leash_pay_payment_link`).

Pass `source: 'leash'` or `source: 'pay-skills'` to scope to a single catalogue;
`'all'` (the default) merges both.

For `pay-skills` items, `slug` carries the provider's fully-qualified name
(e.g. `agentmail/email`). Pass that to `paySkillsProvider(fqn)` to expand a
provider into its individual paid endpoints — the second hop in pay.sh's
`search_skills → get_skill_endpoints → curl` agent flow. The response is a
`PaySkillsProvider` with `endpoints[]`, each carrying an absolute `url`,
`method`, `pricing`, `protocol`, `supported_usd`, and `probe_status`. Hand
`endpoint.url` straight to `@leashmarket/buyer-kit` (or `@leashmarket/mcp`'s
`leash_pay_payment_link`) to settle.

```ts
const search = await leash.discover({ capability: 'email', source: 'pay-skills', limit: 5 });
for (const item of search.items) {
  const provider = await leash.paySkillsProvider(item.slug); // e.g. 'agentmail/email'
  console.log(`${provider.title}: ${provider.endpoints.length} endpoints`);
  // provider.endpoints[0].url is a paid x402 URL; pay it with @leashmarket/buyer-kit.
}
```

### Receipts (legacy API key)

| Method                                        | Endpoint                  |
| --------------------------------------------- | ------------------------- |
| `receipts({ agentMint, direction?, limit? })` | `GET /v1/receipts/{mint}` |

### Payment links (legacy API key)

| Method                         | Endpoint                        |
| ------------------------------ | ------------------------------- |
| `createPaymentLink(input)`     | `POST /v1/payment-links`        |
| `listPaymentLinks(query?)`     | `GET /v1/payment-links`         |
| `getPaymentLink(id)`           | `GET /v1/payment-links/{id}`    |
| `updatePaymentLink(id, patch)` | `PATCH /v1/payment-links/{id}`  |
| `deletePaymentLink(id)`        | `DELETE /v1/payment-links/{id}` |

### Agent webhooks (X-Leash-Sig)

| Method                            | Endpoint                                 |
| --------------------------------- | ---------------------------------------- |
| `createWebhook({ url, events? })` | `POST /v1/agents/{mint}/webhooks`        |
| `listWebhooks()`                  | `GET /v1/agents/{mint}/webhooks`         |
| `deleteWebhook(id)`               | `DELETE /v1/agents/{mint}/webhooks/{id}` |

## Examples

```ts
// Find an OCR provider under a penny, vet, then create a link to expose
// the result downstream.
const found = await pub.discover({ capability: 'ocr', max_price_usdc: 0.01 });
const top = found.items[0];
const rep = await pub.reputation({ agentMint: top.seller_agent_mint! });
if (rep.rating < 0.5) throw new Error('seller too cold');

const link = await keyed.createPaymentLink({
  label: 'Re-tag image',
  owner_agent: 'AjfeyP...',
  price: '$0.005',
  currency: 'USDC',
  response: { status: 200, mimeType: 'application/json', body: { ok: true } },
});
console.log('share', link.share_url);
```

## Error model

Every non-2xx response throws `LeashError`:

```ts
import { LeashError } from '@leashmarket/sdk';

try {
  await keyed.createPaymentLink({
    /* … */
  });
} catch (err) {
  if (err instanceof LeashError) {
    console.log(err.status, err.message, err.body);
  }
}
```

## Manual signing

The signing helpers are exported for callers who want to build their
own transport (workers, gRPC bridge, Postman):

```ts
import { signRequest, buildEnvelope } from '@leashmarket/sdk';
```

The envelope is `SHA-256(method\npath\ntimestamp\nsha256(body)\nagent)`,
exactly what the API verifies.
